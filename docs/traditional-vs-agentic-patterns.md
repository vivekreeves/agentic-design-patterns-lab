# Traditional vs Agentic Patterns

## Traditional Pattern Focus
Traditional software patterns optimize deterministic execution, modularity, and maintainability.

## Agentic Pattern Focus
Agentic patterns manage probabilistic reasoning loops, dynamic tool usage, memory updates, model routing, and uncertainty-aware control.

## Comparison Matrix
- Unit of design: classes/services vs goals/skills/tools/memory
- Control style: fixed workflows vs adaptive workflows
- Failure mode: exceptions and retries vs uncertainty, drift, and misalignment
- Optimization target: throughput and correctness vs quality-latency-cost-risk frontier

## Practical Mapping
- Factory -> agent/tool construction policy
- Strategy -> model and prompt routing policy
- Observer -> runtime telemetry and evaluation hooks
- Circuit breaker -> safety guardrails and escalation pathways
